# Honeypot Login (Phase 2)

A Dockerized login honeypot with a React frontend, Nginx reverse proxy, and Express backend that records login attempts to CSV files.

## Overview

This project intentionally rejects login attempts and stores them for analysis.

- Frontend: React app served by Nginx
- Reverse proxy: Nginx forwards `/api/*` to backend
- Backend: Express API for login capture and retrieval
- Storage: CSV files (no relational database)

## Architecture

- `frontend` container listens on port `3000`
- `backend` container listens on internal port `5000`
- Nginx proxies `http://backend:5000/api/`
- Docker bind mounts persist logs/data on host

## API Endpoints

Base path: `/api/auth`

- `POST /api/auth/login`
  - Body: `{ "username": "...", "password": "..." }`
  - Behavior: always logs attempt, always returns `401` (`failed login`)
- `GET /api/auth/attempts`
  - Returns stored login attempts from `logs/auth_log.csv`

Health:

- `GET /health` (backend)

## Logging

### Backend logs

- `logs/auth_log.csv`: captured login attempts

Backend Logs are application-level logs:
- Generated by Node/Express code
- Backend CSV includes headers and appends records over time.

### Frontend (Nginx) logs

- `logs/access.csv`: Nginx access log in CSV format (edge/reverse proxy)
- `logs/error.log`: Nginx operational errors/warnings

Frontend Logs are edge/server logs:
- Generated by Nginx before/while proxying to backend.
- Capture raw HTTP traffic details (IP, method, URI, status, referer, user-agent, request time).
- Useful for traffic monitoring, suspicious scanning patterns, and proxy-level troubleshooting.
- Nginx does not automatically add a CSV header row for access logs unless you add startup initialization.

## Request/Response Flow

1. Client opens http://localhost:3000 and sends request to Nginx (frontend container).
2. Nginx serves static React files for UI routes and logs request in access.csv.
3. When login is submitted, browser sends POST /api/auth/login to Nginx.
4. Nginx proxies /api/* to backend (http://backend:5000/api/*).
5. auth.routes.js handles /api/auth/login, reads username/password, IP, and User-Agent.
6. authLogger service writes attempt record to auth_log.csv with timestamp and ID.
7. Backend intentionally returns 401 failed login (honeypot behavior).
8. Response goes back through Nginx to client; Nginx logs final status/latency in access.csv.
9. Any proxy/config/runtime issues are written by Nginx to error.log.

## Prerequisites
- Docker Desktop (or Docker Engine + Compose)

## Run with Docker

From repository root:

```bash
docker compose up -d --build
```

Open:

- Frontend: `http://localhost:3000`

Stop:

```bash
docker compose down
```

## Notes

- CSV and log files in `logs/` are runtime artifacts.
- Keep sensitive data handling in mind if this project is used outside lab/testing contexts.
